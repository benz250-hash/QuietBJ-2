QuietBJ API 调用逻辑说明（v516）

一、正式查询时会调用哪些高德 API
1. geocode(query)
- 中文注释：把用户输入的小区 / 楼栋地址转成经纬度
- 说明：这是正式查询的第一步

2. geocode(cleaned_query)【可选】
- 中文注释：如果原始 geocode 失败，就用清洗后的地址再试一次
- 说明：只有第一次 geocode 没结果时才会触发

3. reverse_geocode(location)
- 中文注释：根据楼栋点拿周边道路、结构化地址信息
- 说明：道路底座主要来自这里返回的 roads

4. search_around(location, "便利店 超市 商场 购物服务 生活服务", 300)
- 中文注释：查附近商业 / 生活服务
- 说明：用于商业活跃度影响

5. search_around(location, "餐饮服务", 300)
- 中文注释：查附近餐饮
- 说明：用于餐饮扰动影响

6. search_around(location, "地铁站", 800)
- 中文注释：查附近轨道交通
- 说明：用于轨道影响

二、正式查询时不会再调用什么
1. input_tips(query)
- 中文注释：输入联想候选地址
- 说明：v516 里正式查询已经不再调用它，避免多打一笔 API

2. 高速 / 快速路 / G6 / G7 / 环路 / 高架 的补查 around 搜索
- 中文注释：额外补查可能漏掉的高速候选
- 说明：v516 已删除，回到低 API 逻辑

三、单次正式查询的 API 次数
常见情况：
- geocode 1 次
- reverse_geocode 1 次
- around 搜索 3 次（商业 / 餐饮 / 地铁）
合计：5 次

最坏情况：
- 第一次 geocode 失败，再重试 1 次 cleaned_query
合计：6 次

四、道路底座的当前逻辑
- 只使用 reverse_geocode(...).roads
- 然后在本地把 roads 分成：
  - 高速 / 快速路
  - 主干路
  - 次干路
  - 小路 / 支路
  - 小区内部路
- 每个等级取最近一条进入建议分
